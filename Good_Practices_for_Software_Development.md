# Good Practices for Software Development

### Question 1 - Which point(s) were new to you?

**Answer:**  
- The idea of joining meetings 5-10 minutes early to casually connect with teammates was new.  
- Using sandbox environments like CodeSandbox or CodePen to share the entire setup is something I hadn’t considered before.  
- The emphasis on over-communicating even for issues like internet/laptop problems or minor delays was insightful.  
- The suggestion to compile all questions into a single message instead of sending multiple messages was new and very practical.  
- Using tools like Loom to create screencasts for explaining problems more effectively was something I hadn’t thought of.  
- Tracking time to improve productivity using apps like Boosted or similar tools was a helpful idea.  

### Question 2 - Which area do you think you need to improve on? What are your ideas to make progress in that area?

**Answer:**  
- I need to improve on maintaining 100% involvement and focus during remote work.  
- Ideas to improve:  
  - Block social media apps during work hours using tools like Freedom or TimeLimit.  
  - Keep phone on silent and remove notifications.  
  - Follow a structured work schedule with breaks to maintain concentration.  
  - Track work hours using apps like Boosted to understand productivity patterns.  
  - Ensure proper diet and short exercise sessions to keep energy levels consistent.  
  - Practice deep work techniques, such as time-blocking and focused work sprints.  
  - Set clear daily goals and priorities to avoid distractions.  
  - Regularly review progress and adjust workflow for efficiency.
